# 用户管理系统

## 用户实体模型

### User 实体

**表名**：`users`

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | BIGINT | PK, AUTO_INCREMENT | 用户唯一标识 |
| `username` | VARCHAR(50) | NOT NULL, UNIQUE | 用户名，用于登录与展示 |
| `email` | VARCHAR(100) | NOT NULL, UNIQUE | 电子邮件 |
| `password` | VARCHAR(255) | NOT NULL | BCrypt 加密存储 |
| `avatar_url` | VARCHAR(500) | - | 头像图片 URL |
| `bio` | VARCHAR(500) | - | 个人简介 |
| `created_at` | TIMESTAMP | - | 注册时间（自动生成） |
| `updated_at` | TIMESTAMP | - | 最后更新时间（自动更新） |

**索引**：`idx_username`、`idx_email`

## 认证机制

### 注册流程

```
客户端 POST /api/auth/register
  ↓ RegisterRequest { username, email, password }
UserService.register()
  ├─ 检查用户名/邮箱是否已存在
  ├─ BCrypt 加密密码
  ├─ 保存 User 实体到数据库
  ├─ 生成 JWT Token（有效期 24 小时）
  └─ 返回 AuthResponse { token, user }
```

### 登录流程

```
客户端 POST /api/auth/login
  ↓ LoginRequest { username, password }
UserService.login()
  ├─ 通过 AuthenticationManager 验证用户名/密码
  ├─ BCrypt 比对密码
  ├─ 生成 JWT Token
  └─ 返回 AuthResponse { token, user }
```

### JWT Token 规格

| 属性 | 值 |
|------|---|
| 算法 | HS256（HMAC SHA-256） |
| 有效期 | 86400000ms（24 小时） |
| Payload | `sub`（username）、`iat`（签发时间）、`exp`（过期时间） |
| 请求头格式 | `Authorization: Bearer <token>` |

### 获取当前用户

```
GET /api/auth/me
Authorization: Bearer <token>
  ↓
JwtAuthenticationFilter 解析 Token，填充 SecurityContext
  ↓
@AuthenticationPrincipal UserDetails → 查询数据库 → 返回 UserDto
```

## 用户资料管理

### 查看公开资料

```
GET /api/users/{id}
```

返回用户公开信息（不含密码），包括：`id`、`username`、`avatarUrl`、`bio`、`createdAt`

### 更新个人资料

```
PUT /api/dashboard/profile
Authorization: Bearer <token>

UpdateProfileRequest {
  username: string (可选),
  bio: string (可选)
}
```

### 上传头像

```
POST /api/dashboard/avatar
Authorization: Bearer <token>
Content-Type: multipart/form-data

file: <图片文件>
```

头像上传到 `{upload.path}/avatars/` 目录，URL 格式：`/uploads/avatars/{uuid}.{ext}`

## 用户服务层（UserService）

核心方法：

| 方法 | 说明 |
|------|------|
| `register(RegisterRequest)` | 用户注册，返回 AuthResponse |
| `login(LoginRequest)` | 用户登录，返回 AuthResponse |
| `getCurrentUser(username)` | 根据用户名获取当前用户 UserDto |
| `getUserById(id)` | 根据 ID 获取用户公开信息 |

---

## 密码安全策略

- 使用 **BCryptPasswordEncoder** 加密存储，cost factor 默认为 10
- 密码不可逆，忘记密码需重置（当前版本未实现重置功能）
- JWT Token 存储在客户端 `localStorage`，登出时客户端主动清除

## 前端认证状态管理（auth store）

使用 Pinia 管理认证状态：

```typescript
// stores/auth.ts
const token = ref<string | null>(localStorage.getItem('token'))
const user = ref<User | null>(JSON.parse(localStorage.getItem('user') || 'null'))
const isAuthenticated = computed(() => !!token.value)
```

**页面刷新持久化**：Token 与用户信息存储在 `localStorage`，页面刷新后自动恢复登录状态。

**路由守卫**：
- 已认证用户访问 `/login`、`/register` 时自动重定向到首页
- 未认证用户访问需要认证的页面时重定向到登录页，并携带 `redirect` 参数
