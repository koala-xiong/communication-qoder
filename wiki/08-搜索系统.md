# 搜索系统

## 搜索模块概览

搜索系统提供三类搜索能力：

| 搜索类型 | 接口 | 说明 |
|---------|------|------|
| 内容搜索 | `GET /api/search/contents` | 按关键词或标签搜索已发布内容 |
| 用户搜索 | `GET /api/search/users` | 按用户名搜索用户 |
| 标签搜索 | `GET /api/search/tags/popular` | 获取热门标签 |
| 标签建议 | `GET /api/search/tags/suggest` | 标签自动补全 |

所有搜索接口均为公开接口，无需认证。

---

## 内容搜索

### 接口

```
GET /api/search/contents?q=<关键词>&tag=<标签>&page=0&size=10
```

**参数说明**：

| 参数 | 是否必填 | 说明 |
|------|---------|------|
| `q` | 否 | 搜索关键词，匹配标题和正文 |
| `tag` | 否 | 标签名，精确匹配 |
| `page` | 否 | 页码，默认 0 |
| `size` | 否 | 每页条数，默认 10 |

`q` 和 `tag` 可单独使用，也可组合使用（同时满足）。

### 搜索实现

**关键词搜索**（`q` 参数）：

利用 MySQL 全文索引对 `contents` 表的 `title` 和 `body` 字段进行搜索：

```sql
-- 全文索引搜索（FULLTEXT INDEX）
SELECT * FROM contents
WHERE MATCH(title, body) AGAINST (:keyword IN BOOLEAN MODE)
  AND status = 'PUBLISHED'
ORDER BY created_at DESC
```

或基于 `LIKE` 模糊匹配（兜底方案）：

```sql
SELECT * FROM contents
WHERE (title LIKE %:keyword% OR body LIKE %:keyword%)
  AND status = 'PUBLISHED'
ORDER BY created_at DESC
```

**标签搜索**（`tag` 参数）：

```sql
SELECT DISTINCT c.*
FROM contents c
JOIN content_tags ct ON c.id = ct.content_id
WHERE ct.tag = :tag
  AND c.status = 'PUBLISHED'
ORDER BY c.created_at DESC
```

### 返回结构

```json
{
  "content": [ { ...ContentDto } ],
  "totalElements": 42,
  "totalPages": 5,
  "page": 0,
  "size": 10
}
```

## 用户搜索

### 接口

```
GET /api/search/users?q=<关键词>&page=0&size=10
```

**搜索逻辑**：模糊匹配用户名（`username LIKE %:q%`），返回 `PageResponse<UserDto>`。

### 返回结构

```json
{
  "content": [
    {
      "id": 1,
      "username": "alice",
      "avatarUrl": "/uploads/avatars/xxx.jpg",
      "bio": "内容创作者"
    }
  ],
  "totalElements": 3,
  "totalPages": 1,
  "page": 0,
  "size": 10
}
```

## 热门标签

### 接口

```
GET /api/search/tags/popular?limit=20
```

统计 `content_tags` 表中标签出现频次，返回频次最高的前 N 个标签：

```sql
SELECT tag, COUNT(*) as count
FROM content_tags
GROUP BY tag
ORDER BY count DESC
LIMIT :limit
```

**返回**：`string[]`，例如 `["技术", "生活", "摄影", "旅行"]`

---

## 标签自动补全

### 接口

```
GET /api/search/tags/suggest?q=<前缀>
```

返回以指定前缀开头的标签列表：

```sql
SELECT DISTINCT tag
FROM content_tags
WHERE tag LIKE :prefix%
LIMIT 10
```

**返回**：`string[]`

## 前端搜索功能

### 全局搜索框（AppHeader）

- 位于顶部导航栏右侧
- 输入时防抖处理（300ms），减少不必要的请求
- 按下 Enter 或点击搜索按钮跳转到 `SearchView`

### 搜索结果页（SearchView）

- URL 格式：`/search?q=<关键词>&type=content|user`
- Tab 切换：内容搜索 / 用户搜索
- 关键词变化时重新发起请求
- 搜索结果为空时展示 `EmptyState` 组件

### 热门标签展示

- 在搜索页展示热门标签列表
- 点击标签触发标签搜索（`?tag=<标签名>`）
- 在内容创建页面输入标签时调用 `suggest` 接口提供自动补全
