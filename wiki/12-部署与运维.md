# 部署与运维

## 容器化部署

项目使用 Docker Compose 编排三个服务：

```
┌───────────────┐
│   frontend    │  ← Nginx 静态服务
│   (port 80)   │
└───────┬───────┘
        │
        ▼
┌───────────────┐
│    backend    │  ← Spring Boot
│  (port 8080)  │
└───────┬───────┘
        │
        ▼
┌───────────────┐
│     mysql     │  ← MySQL 8.0
│  (port 3306)  │
└───────────────┘
```

### docker-compose.yml 配置

```yaml
version: '3.8'

services:
  mysql:
    image: mysql:8.0
    container_name: communication-mysql
    restart: unless-stopped
    environment:
      MYSQL_ROOT_PASSWORD: root123
      MYSQL_DATABASE: communication
      MYSQL_USER: communication
      MYSQL_PASSWORD: comm123
    ports:
      - "3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql
      - ./init.sql:/docker-entrypoint-initdb.d/init.sql:ro
    command: --default-authentication-plugin=mysql_native_password --character-set-server=utf8mb4 --collation-server=utf8mb4_unicode_ci
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 10s
      timeout: 5s
      retries: 5

  backend:
    build:
      context: ./communication-backend
      dockerfile: Dockerfile
    container_name: communication-backend
    restart: unless-stopped
    environment:
      SPRING_PROFILES_ACTIVE: docker
      SPRING_DATASOURCE_URL: jdbc:mysql://mysql:3306/communication?useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=UTC
      SPRING_DATASOURCE_USERNAME: communication
      SPRING_DATASOURCE_PASSWORD: comm123
      JWT_SECRET: your-super-secret-jwt-key-change-in-production
      UPLOAD_PATH: /app/uploads
    ports:
      - "8080:8080"
    volumes:
      - uploads_data:/app/uploads
    depends_on:
      mysql:
        condition: service_healthy

  frontend:
    build:
      context: ./communication-frontend
      dockerfile: Dockerfile
    container_name: communication-frontend
    restart: unless-stopped
    ports:
      - "80:80"
    depends_on:
      - backend

volumes:
  mysql_data:
  uploads_data:
```

### 服务依赖与健康检查

- **MySQL**：通过 `healthcheck` 检测就绪状态
- **Backend**：等待 MySQL `healthy` 后启动
- **Frontend**：等待 Backend 启动

### 数据卷

| 卷名 | 用途 | 挂载路径 |
|------|------|---------|
| `mysql_data` | MySQL 数据持久化 | `/var/lib/mysql` |
| `uploads_data` | 上传文件持久化 | `/app/uploads` |

## 环境配置管理

### 后端多环境配置

| 配置文件 | 用途 | 激活方式 |
|---------|------|---------|
| `application.yml` | 本地开发默认配置 | 默认 |
| `application-docker.yml` | Docker 容器环境 | `SPRING_PROFILES_ACTIVE=docker` |
| `application-h2.yml` | H2 内存数据库（测试）| `SPRING_PROFILES_ACTIVE=h2` |
| `application-test.yml` | 测试环境 | 测试时自动激活 |

### 关键环境变量

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `SPRING_DATASOURCE_URL` | 数据库连接 URL | `jdbc:mysql://localhost:3306/communication` |
| `SPRING_DATASOURCE_USERNAME` | 数据库用户名 | `root` |
| `SPRING_DATASOURCE_PASSWORD` | 数据库密码 | `root` |
| `JWT_SECRET` | JWT 签名密钥 | 硬编码默认值（生产环境必须覆盖） |
| `UPLOAD_PATH` | 文件上传目录 | `./uploads` |

### 生产环境建议

```bash
# 生成强密钥
openssl rand -base64 32

# 设置环境变量
export JWT_SECRET="your-production-secret-key-must-be-at-least-32-chars"
export SPRING_DATASOURCE_PASSWORD="strong-db-password"
```

## 前后端集成部署

### 前端 Nginx 配置

前端 Dockerfile 使用 Nginx 作为静态资源服务器，配置如下：

```nginx
server {
    listen 80;
    server_name localhost;

    location / {
        root /usr/share/nginx/html;
        try_files $uri $uri/ /index.html;
    }

    location /api {
        proxy_pass http://backend:8080/api;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /uploads {
        proxy_pass http://backend:8080/uploads;
    }
}
```

### 部署步骤

```bash
# 1. 构建并启动所有服务
docker-compose up -d --build

# 2. 查看服务状态
docker-compose ps

# 3. 查看日志
docker-compose logs -f backend
docker-compose logs -f frontend
docker-compose logs -f mysql

# 4. 停止服务
docker-compose down

# 5. 完全清理（包括数据卷）
docker-compose down -v
```

## 监控与日志管理

### 日志查看

```bash
# 实时查看后端日志
docker logs -f communication-backend

# 查看最近 100 行日志
docker logs --tail 100 communication-backend

# 查看带时间戳的日志
docker logs -t communication-backend
```

### 日志配置

Spring Boot 日志配置在 `application.yml`：

```yaml
logging:
  level:
    com.communication: INFO
    org.springframework.security: WARN
```

### 健康检查端点

Spring Boot Actuator 默认提供：

```
GET /actuator/health    # 应用健康状态
GET /actuator/info      # 应用信息
```

## 备份与恢复

### 数据库备份

```bash
# 备份 MySQL 数据
docker exec communication-mysql mysqldump -u root -proot123 communication > backup.sql

# 恢复数据
docker exec -i communication-mysql mysql -u root -proot123 communication < backup.sql
```

### 上传文件备份

```bash
# 备份上传文件
docker run --rm -v communication_uploads_data:/data -v $(pwd):/backup alpine tar czf /backup/uploads-backup.tar.gz -C /data .

# 恢复上传文件
docker run --rm -v communication_uploads_data:/data -v $(pwd):/backup alpine tar xzf /backup/uploads-backup.tar.gz -C /data
```

## 扩容与性能优化

### 水平扩展

当前架构为单实例部署，如需水平扩展：

1. **后端**：使用负载均衡器（Nginx/HAProxy）分发请求到多个后端实例
2. **数据库**：主从复制 + 读写分离，或使用云数据库服务
3. **文件存储**：迁移到对象存储（如 AWS S3、阿里云 OSS）

### 性能监控建议

- 使用 Prometheus + Grafana 监控应用指标
- 使用 ELK Stack 集中收集和分析日志
- 数据库慢查询日志分析优化
