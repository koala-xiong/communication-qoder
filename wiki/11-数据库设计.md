# 数据库设计

## 概览

数据库使用 **MySQL 8.0**，字符集 `utf8mb4`，Collation `utf8mb4_unicode_ci`，支持完整的 Emoji 和多语言字符。

数据库迁移通过 **Flyway** 管理，迁移脚本位于：

```
communication-backend/src/main/resources/db/migration/
├── V1__init_users.sql              # 用户表
├── V2__create_contents.sql         # 内容表
├── V3__create_comments_subscriptions.sql   # 评论表 + 订阅表
└── V4__create_content_tags.sql     # 内容标签表
```

## 表结构详细说明

### users 表（V1）

```sql
CREATE TABLE users (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    username    VARCHAR(50)  NOT NULL UNIQUE,
    email       VARCHAR(100) NOT NULL UNIQUE,
    password    VARCHAR(255) NOT NULL,
    avatar_url  VARCHAR(500),
    bio         VARCHAR(500),
    created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_username (username),
    INDEX idx_email (email)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

**设计说明**：
- `password` 存储 BCrypt 哈希（固定 60 字符，预留 255 字符）
- `username` 和 `email` 建立唯一索引，防重复并加速查询

### contents 表（V2）

```sql
CREATE TABLE contents (
    id            BIGINT AUTO_INCREMENT PRIMARY KEY,
    author_id     BIGINT NOT NULL,
    title         VARCHAR(200) NOT NULL,
    body          TEXT,
    media_url     VARCHAR(500),
    media_type    ENUM('TEXT', 'IMAGE', 'VIDEO') NOT NULL DEFAULT 'TEXT',
    view_count    INT DEFAULT 0,
    comment_count INT DEFAULT 0,
    status        ENUM('DRAFT', 'PUBLISHED') NOT NULL DEFAULT 'PUBLISHED',
    created_at    TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at    TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (author_id) REFERENCES users(id) ON DELETE CASCADE,
    INDEX idx_author_id (author_id),
    INDEX idx_status (status),
    INDEX idx_created_at (created_at DESC),
    FULLTEXT INDEX idx_fulltext (title, body)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

**设计说明**：
- `FULLTEXT INDEX` 支持全文搜索，使用 `MATCH ... AGAINST` 语法
- `comment_count` 冗余字段，避免 `COUNT(*)` 查询，提升列表接口性能
- `view_count` 同上，直接读取，无需 JOIN
- `status` 使用 ENUM 限制值域，DRAFT 草稿不对外展示

### comments 表（V3）

```sql
CREATE TABLE comments (
    id         BIGINT AUTO_INCREMENT PRIMARY KEY,
    content_id BIGINT NOT NULL,
    user_id    BIGINT NOT NULL,
    body       TEXT NOT NULL,
    parent_id  BIGINT DEFAULT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (content_id) REFERENCES contents(id) ON DELETE CASCADE,
    FOREIGN KEY (user_id)    REFERENCES users(id)    ON DELETE CASCADE,
    FOREIGN KEY (parent_id)  REFERENCES comments(id) ON DELETE CASCADE,
    INDEX idx_comments_content (content_id),
    INDEX idx_comments_user    (user_id),
    INDEX idx_comments_parent  (parent_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

**设计说明**：
- `parent_id` 自引用，实现嵌套评论（树形结构）
- 三个 FK 均设置 `ON DELETE CASCADE`，防止孤儿数据

### subscriptions 表（V3）

```sql
CREATE TABLE subscriptions (
    id            BIGINT AUTO_INCREMENT PRIMARY KEY,
    subscriber_id BIGINT NOT NULL,
    author_id     BIGINT NOT NULL,
    created_at    TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (subscriber_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (author_id)     REFERENCES users(id) ON DELETE CASCADE,
    UNIQUE KEY uk_subscription (subscriber_id, author_id),
    INDEX idx_subscriptions_subscriber (subscriber_id),
    INDEX idx_subscriptions_author     (author_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

**设计说明**：
- `UNIQUE KEY uk_subscription` 防止重复关注
- 双向索引支持「我的订阅」和「我的粉丝」两个查询方向

### content_tags 表（V4）

```sql
CREATE TABLE content_tags (
    id         BIGINT AUTO_INCREMENT PRIMARY KEY,
    content_id BIGINT NOT NULL,
    tag        VARCHAR(50) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (content_id) REFERENCES contents(id) ON DELETE CASCADE,
    INDEX idx_content_tags_content (content_id),
    INDEX idx_content_tags_tag     (tag)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

**设计说明**：
- 采用独立行存储每个标签（非 JSON 或逗号分隔），便于按标签搜索和统计
- `idx_content_tags_tag` 支持热门标签聚合查询

## 表关系设计

| 关系 | 类型 | 实现方式 |
|------|------|---------|
| 用户 → 内容 | 一对多 | `contents.author_id → users.id` |
| 内容 → 评论 | 一对多 | `comments.content_id → contents.id` |
| 用户 → 评论 | 一对多 | `comments.user_id → users.id` |
| 评论 → 评论（回复）| 自引用一对多 | `comments.parent_id → comments.id` |
| 内容 → 标签 | 一对多 | `content_tags.content_id → contents.id` |
| 用户 ↔ 用户（订阅）| 多对多 | `subscriptions` 关联表 |

## 数据库迁移策略

### Flyway 配置

```yaml
spring:
  flyway:
    enabled: true
    locations: classpath:db/migration
    baseline-on-migrate: true
```

### 迁移版本规则

| 版本 | 文件名 | 说明 |
|------|--------|------|
| V1 | `V1__init_users.sql` | 用户表初始化 |
| V2 | `V2__create_contents.sql` | 内容表创建 |
| V3 | `V3__create_comments_subscriptions.sql` | 评论与订阅表 |
| V4 | `V4__create_content_tags.sql` | 标签表 + 搜索索引 |

**新增迁移原则**：
1. 文件名格式 `V{n}__{description}.sql`，版本号递增
2. 已执行的迁移文件不可修改（Flyway 会校验 checksum）
3. 回滚需单独编写 undo 脚本（社区版不支持自动回滚）

## 性能优化设计

### 已有索引

- 用户名/邮箱唯一索引（注册时重复校验）
- 内容时间降序索引（首页 Feed 查询）
- 内容作者索引（作者内容列表）
- 评论内容/父评论索引（评论列表/嵌套评论）
- 订阅者/作者双向索引（关系查询）
- 标签索引（标签搜索、热门标签聚合）
- 全文索引（内容关键词搜索）

### 冗余计数字段

`contents.view_count` 和 `contents.comment_count` 为冗余存储：
- 避免每次列表查询都执行 `COUNT(*)` JOIN
- 代价是需要在 Service 层手动维护同步

### 注意事项

- `view_count` 高频更新场景下可考虑使用 Redis 缓存 + 定时写回
- 大数据量下订阅 Feed 查询（`IN` 子查询）可能成为瓶颈，可考虑引入 Elasticsearch 或消息队列预生成 Feed
