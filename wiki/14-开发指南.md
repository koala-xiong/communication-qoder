# 开发指南

## 代码规范

### Java 代码规范

- **命名规范**：
  - 类名：PascalCase（如 `UserService`）
  - 方法名/变量名：camelCase（如 `getUserById`）
  - 常量：SCREAMING_SNAKE_CASE

- **包结构**：
  ```
  com.communication
  ├── config/       # 配置类
  ├── controller/   # 控制器
  ├── service/      # 服务层
  ├── repository/   # 数据访问层
  ├── entity/       # 实体类
  ├── dto/          # 数据传输对象
  ├── exception/    # 异常类
  └── util/         # 工具类
  ```

- **代码风格**：
  - 使用 4 空格缩进
  - 类注释说明职责
  - 方法注释说明参数和返回值

### TypeScript/Vue 代码规范

- **命名规范**：
  - 组件名：PascalCase（如 `ContentCard.vue`）
  - 组合式函数：camelCase 前缀 `use`（如 `useAuthStore`）
  - 类型/接口：PascalCase（如 `UserDto`）

- **文件组织**：
  ```
  src/
  ├── api/          # API 请求
  ├── components/   # 组件
  ├── views/        # 页面视图
  ├── stores/       # Pinia stores
  ├── router/       # 路由配置
  └── styles/       # 样式文件
  ```

- **Vue 单文件组件**：
  - 使用 `<script setup>` 语法
  - 使用 TypeScript（`lang="ts"`）
  - 组件名与文件名一致

## 构建配置

### 后端构建配置

**Maven Wrapper**：

```bash
# 项目已包含 mvnw，无需本地安装 Maven
./mvnw clean install
./mvnw spring-boot:run
```

**pom.xml 关键依赖**：

```xml
<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>3.2.0</version>
</parent>

<dependencies>
    <!-- Spring Boot Starters -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-security</artifactId>
    </dependency>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-data-jpa</artifactId>
    </dependency>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-validation</artifactId>
    </dependency>
    
    <!-- Database -->
    <dependency>
        <groupId>mysql</groupId>
        <artifactId>mysql-connector-java</artifactId>
    </dependency>
    <dependency>
        <groupId>org.flywaydb</groupId>
        <artifactId>flyway-mysql</artifactId>
    </dependency>
    
    <!-- JWT -->
    <dependency>
        <groupId>io.jsonwebtoken</groupId>

    <!-- Testing -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-test</artifactId>
        <scope>test</scope>
    </dependency>
</dependencies>
```

### 前端构建配置

**Vite 配置**（`vite.config.ts`）：

```typescript
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import { resolve } from 'path'

export default defineConfig({
  plugins: [vue()],
  resolve: {
    alias: {
      '@': resolve(__dirname, 'src')
    }
  },
  server: {
    port: 5173,
    proxy: {
      '/api': {
        target: 'http://localhost:8080',
        changeOrigin: true
      }
    }
  }
})
```

**package.json 脚本**：

```json
{
  "scripts": {
    "dev": "vite",
    "build": "vue-tsc && vite build",
    "preview": "vite preview",
    "test:unit": "vitest",
    "test:e2e": "playwright test"
  }
}
```

### Docker 构建配置

**后端 Dockerfile**：

```dockerfile
FROM eclipse-temurin:21-jdk-alpine AS build
WORKDIR /app
COPY . .
RUN ./mvnw clean package -DskipTests

FROM eclipse-temurin:21-jre-alpine
WORKDIR /app
COPY --from=build /app/target/*.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]
```

**前端 Dockerfile**：

```dockerfile
FROM node:20-alpine AS build
WORKDIR /app
COPY package.json pnpm-lock.yaml ./
RUN npm install -g pnpm && pnpm install
COPY . .
RUN pnpm build

FROM nginx:alpine
COPY --from=build /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
```

## 开发流程

### 功能开发流程

```
1. 创建功能分支
   git checkout -b feature/xxx

2. 编写代码
   - 后端：Entity → Repository → Service → Controller
   - 前端：API → Store → Component → View

3. 编写测试
   - 单元测试覆盖核心业务逻辑

4. 本地验证
   - 启动后端：./mvnw spring-boot:run
   - 启动前端：pnpm dev
   - 手动验证功能

5. 提交代码
   git add .
   git commit -m "feat: xxx"
   git push origin feature/xxx

6. 创建 Pull Request
   - 填写 PR 描述
   - 关联 Issue（如有）
   - 请求 Code Review

7. 合并到主分支
```

### 数据库变更流程

```
1. 创建新的 Flyway 迁移脚本
   V{n}__{description}.sql

2. 编写 SQL 变更（CREATE TABLE / ALTER TABLE / CREATE INDEX）

3. 本地测试
   - 启动应用，Flyway 自动执行迁移
   - 验证数据库结构正确

4. 提交迁移脚本
   - 迁移脚本必须与代码变更一起提交
   - 已执行的脚本不可修改
```

## 调试与故障排除

### 后端调试

**本地调试**：

```bash
# 使用 IDEA 调试模式启动
# 或在命令行添加调试参数
./mvnw spring-boot:run -Dspring-boot.run.jvmArguments="-agentlib:jdwp=transport=dt_socket,server=y,suspend=n,address=5005"
```

**日志级别调整**：

```yaml
logging:
  level:
    com.communication: DEBUG
    org.springframework.security: DEBUG
```

### 前端调试

**浏览器 DevTools**：
- Vue DevTools 插件查看组件状态和 Pinia store
- Network 面板查看 API 请求
- Console 查看日志和错误

**Vite HMR**：
- 修改代码后自动热更新
- 状态丢失时使用 `preserveState` 或手动保存

### 常见问题排查

| 问题 | 排查步骤 |
|------|---------|
| 后端启动失败 | 检查数据库连接、端口占用、依赖下载 |
| 前端无法连接后端 | 检查 CORS 配置、代理配置、后端是否启动 |
| 数据库迁移失败 | 检查 Flyway 版本冲突、SQL 语法错误 |
| 401 认证失败 | 检查 JWT Token、Token 是否过期、请求头格式 |
| 文件上传失败 | 检查上传目录权限、文件大小限制 |

## Git 工作流

### 分支策略

```
main                    # 生产分支，稳定版本
  └── develop           # 开发分支，集成测试
       ├── feature/xxx  # 功能分支
       └── bugfix/xxx   # 修复分支
```

### 提交规范

使用 **Conventional Commits** 格式：

```
<type>(<scope>): <subject>

<body>

<footer>
```

**类型（type）**：

| 类型 | 说明 |
|------|------|
| `feat` | 新功能 |
| `fix` | 修复 Bug |
| `docs` | 文档更新 |
| `style` | 代码格式调整（不影响功能）|
| `refactor` | 重构（非 feat/fix）|
| `test` | 测试相关 |
| `chore` | 构建/工具相关 |

**示例**：

```
feat(auth): 添加用户注册功能

- 实现注册接口
- 添加邮箱唯一性校验
- 发送欢迎邮件

Closes #123
```
