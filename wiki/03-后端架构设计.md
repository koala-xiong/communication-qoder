# 后端架构设计

## 整体架构

后端采用经典 MVC 分层架构，基于 Spring Boot 3.2 构建：

```
请求入口
  ↓
JwtAuthenticationFilter（JWT 鉴权过滤器）
  ↓
Controller 层（请求路由与参数校验）
  ↓
Service 层（业务逻辑）
  ↓
Repository 层（数据访问，JPA）
  ↓
Entity 层（数据库实体映射）
  ↓
MySQL 8.0
```

## 应用启动与配置

### 入口类

`CommunicationApplication.java` 是 Spring Boot 启动入口，使用标准 `@SpringBootApplication` 注解。

### 配置文件

| 文件 | 用途 |
|------|------|
| `application.yml` | 主配置（本地开发） |
| `application-docker.yml` | Docker 容器环境配置 |
| `application-h2.yml` | H2 内存数据库（测试环境） |
| `application-test.yml` | 测试专用配置 |

通过 `SPRING_PROFILES_ACTIVE` 环境变量切换 Profile，Docker 中默认激活 `docker` Profile。

## 安全架构设计

### Spring Security 配置（SecurityConfig）

安全配置采用无状态（Stateless）Session 策略，基于 JWT Token 实现认证。

**公开端点（无需认证）**：

| 路径 | 方法 | 说明 |
|------|------|------|
| `/api/auth/**` | ALL | 注册、登录 |
| `/api/contents/**` | GET | 浏览内容 |
| `/api/users/**` | GET | 查看用户信息 |
| `/api/search/**` | GET | 搜索功能 |
| `/api/subscriptions/followers/**` | GET | 查看粉丝列表 |
| `/api/subscriptions/count/**` | GET | 查看订阅数量 |
| `/uploads/**` | GET | 访问上传文件 |

**所有其他请求均需要 JWT 认证。**

**核心 Bean 配置**：

```java
// BCrypt 密码加密器
@Bean
public PasswordEncoder passwordEncoder() {
    return new BCryptPasswordEncoder();
}

// 基于 UserDetailsService 的认证提供器
@Bean
public AuthenticationProvider authenticationProvider() {
    DaoAuthenticationProvider provider = new DaoAuthenticationProvider();
    provider.setUserDetailsService(userDetailsService());
    provider.setPasswordEncoder(passwordEncoder());
    return provider;
}
```

### JWT 认证过滤器（JwtAuthenticationFilter）

继承 `OncePerRequestFilter`，每次请求只执行一次。

**工作流程**：

1. 从 `Authorization` 请求头提取 `Bearer <token>`
2. 调用 `JwtUtil.extractUsername()` 解析用户名
3. 加载 `UserDetails` 并调用 `JwtUtil.isTokenValid()` 验证 Token
4. 验证通过则将 `UsernamePasswordAuthenticationToken` 写入 `SecurityContextHolder`
5. Token 无效时静默放行（不认证），由后续权限校验拦截

## MVC 架构设计

### Controller 层

负责接收 HTTP 请求、参数校验（`@Valid`）、调用 Service 并返回标准响应。

| Controller | 路径前缀 | 职责 |
|-----------|---------|------|
| `AuthController` | `/api/auth` | 注册、登录、获取当前用户 |
| `ContentController` | `/api/contents` | 内容 CRUD、分页查询 |
| `CommentController` | `/api/contents/{id}/comments` | 评论 CRUD |
| `SubscriptionController` | `/api/subscriptions` | 订阅关系管理、订阅 Feed |
| `SearchController` | `/api/search` | 内容/用户/标签搜索 |
| `DashboardController` | `/api/dashboard` | 作者后台统计与管理 |
| `UploadController` | `/api/upload` | 媒体文件上传 |
| `UserController` | `/api/users` | 用户公开信息查询 |

所有响应统一封装为 `ApiResponse<T>`：

```json
{
  "success": true,
  "message": "操作成功",
  "data": { ... }
}
```

### Service 层

封装核心业务逻辑，与 Repository 解耦。

| Service | 职责 |
|---------|------|
| `UserService` | 用户注册/登录、JWT 生成、资料管理 |
| `ContentService` | 内容创建/更新/删除、分页查询、浏览量计数 |
| `CommentService` | 评论发表/删除、嵌套评论查询 |
| `SubscriptionService` | 订阅关系管理、Feed 流聚合 |
| `SearchService` | 内容/用户/标签搜索、热门标签 |
| `DashboardService` | 统计数据聚合、资料更新、头像管理 |
| `FileUploadService` | 文件类型校验、本地存储、URL 生成 |

### Repository 层

基于 Spring Data JPA，通过接口方法名约定或 JPQL 实现查询：

| Repository | 实体 | 关键查询 |
|-----------|------|---------|
| `UserRepository` | `User` | `findByUsername`, `findByEmail` |
| `ContentRepository` | `Content` | 按作者分页、按状态过滤、全文搜索 |
| `CommentRepository` | `Comment` | 按内容分页、按父评论查询 |
| `SubscriptionRepository` | `Subscription` | 订阅关系查询、订阅数统计 |
| `ContentTagRepository` | `ContentTag` | 按标签查询、热门标签统计 |

### DTO 模式设计

项目使用独立 DTO 类隔离 API 层与实体层：

| DTO | 用途 |
|-----|------|
| `LoginRequest` / `RegisterRequest` | 认证请求 |
| `AuthResponse` | 登录/注册响应（含 Token + 用户信息） |
| `UserDto` | 用户公开信息响应 |
| `ContentDto` | 内容详情响应 |
| `CommentDto` | 评论响应 |
| `CreateContentRequest` / `UpdateContentRequest` | 内容创建/更新请求 |
| `CreateCommentRequest` | 评论创建请求 |
| `UpdateProfileRequest` | 资料更新请求 |
| `PageResponse<T>` | 通用分页响应包装 |
| `DashboardStatsDto` | 作者后台统计数据 |
| `SubscriptionCountDto` | 订阅/粉丝数量 |

## 异常处理机制

项目通过 `@ControllerAdvice` 实现全局异常处理，将业务异常统一转换为标准错误响应：

```json
{
  "success": false,
  "code": 404,
  "message": "资源未找到",
  "data": null
}
```

常见异常类型：

- `ResourceNotFoundException`：资源不存在（404）
- `UnauthorizedException`：无权限操作（403）
- `MethodArgumentNotValidException`：请求参数校验失败（400）

## Web 配置

### CORS 配置（CorsConfig）

允许前端开发地址（`http://localhost:5173`）及生产地址跨域请求，支持所有 HTTP 方法与常用请求头。

### 静态资源访问（WebConfig）

将 `/uploads/**` 路径映射到本地 `upload.path` 目录，使上传的图片/视频可直接通过 URL 访问。

### 文件上传限制

```yaml
spring:
  servlet:
    multipart:
      max-file-size: 100MB
      max-request-size: 100MB
```
