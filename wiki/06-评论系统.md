# 评论系统

## 评论实体模型

### Comment 实体

**表名**：`comments`

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | BIGINT | PK, AUTO_INCREMENT | 评论唯一标识 |
| `content_id` | BIGINT | FK → contents.id | 所属内容 |
| `user_id` | BIGINT | FK → users.id | 评论者 |
| `body` | TEXT | NOT NULL | 评论正文 |
| `parent_id` | BIGINT | FK → comments.id, 可为空 | 父评论 ID（用于嵌套回复） |
| `created_at` | TIMESTAMP | - | 创建时间 |
| `updated_at` | TIMESTAMP | - | 更新时间 |

**索引**：
- `idx_comments_content (content_id)`：按内容查询评论
- `idx_comments_user (user_id)`：按用户查询评论
- `idx_comments_parent (parent_id)`：按父评论查询回复

**级联规则**：内容删除时，关联评论自动级联删除（`ON DELETE CASCADE`）

---

## 嵌套评论结构

系统支持**两级评论**（顶级评论 + 回复）：

```
顶级评论（parent_id = NULL）
  ├─ 回复 A（parent_id = 顶级评论 ID）
  ├─ 回复 B（parent_id = 顶级评论 ID）
  └─ 回复 C（parent_id = 顶级评论 ID）
```

- `parent` 字段为 `null` 时表示顶级评论
- `replies` 为该评论的所有直接回复列表
- 删除父评论时，子回复自动级联删除

## API 接口

### 发表评论

```
POST /api/contents/{contentId}/comments
Authorization: Bearer <token>

CreateCommentRequest {
  body: string (必填)
  parentId: Long (可选，回复某条评论时传入)
}
```

**流程**：
1. 验证内容存在
2. 如有 `parentId`，验证父评论存在且属于同一内容
3. 创建 `Comment` 实体，关联内容、用户、父评论
4. 更新 `contents.comment_count`（+1）
5. 返回 `CommentDto`

### 获取评论列表

```
GET /api/contents/{contentId}/comments?page=0&size=20
```

- 只返回顶级评论（`parent_id IS NULL`）
- 每条评论包含其 `replies` 列表
- 按 `created_at ASC` 排序（时间正序）
- 支持分页，默认每页 20 条

### 获取单条评论

```
GET /api/contents/{contentId}/comments/{commentId}
```

### 删除评论

```
DELETE /api/contents/{contentId}/comments/{commentId}
Authorization: Bearer <token>
```

**权限校验**：只有评论作者才能删除，删除后 `comment_count` 自动减少。

## 前端评论组件

### CommentList 组件

- 展示内容的所有顶级评论列表
- 每条评论展开显示回复列表
- 登录用户可见删除按钮（仅限自己的评论）

### CommentInput 组件

- 顶部输入框用于发表顶级评论
- 点击「回复」按钮时，输入框切换为回复模式，传入 `parentId`
- 发送后刷新评论列表

### CommentItem 组件

- 展示单条评论：头像、用户名、内容、时间
- 操作：回复、删除（权限控制）
- 递归展示二级回复

## 评论数同步

`contents.comment_count` 字段通过 Service 层手动维护：
- 创建评论时：`contentService.incrementCommentCount(contentId)`
- 删除评论时：`contentService.decrementCommentCount(contentId)`

> 注意：删除父评论时子评论级联删除，但 `comment_count` 仅减 1（父评论），子评论的数量不单独扣减，实际评论数可能存在偏差。建议后续版本通过数据库触发器或定时任务同步修正。
