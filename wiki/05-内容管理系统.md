# 内容管理系统

## 内容模型设计

### Content 实体

**表名**：`contents`

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | BIGINT | PK, AUTO_INCREMENT | 内容唯一标识 |
| `author_id` | BIGINT | FK → users.id | 作者 ID |
| `title` | VARCHAR(200) | NOT NULL | 内容标题 |
| `body` | TEXT | - | 正文内容 |
| `media_url` | VARCHAR(500) | - | 媒体文件 URL |
| `media_type` | ENUM | NOT NULL | 媒体类型：TEXT/IMAGE/VIDEO |
| `view_count` | INT | DEFAULT 0 | 浏览量 |
| `comment_count` | INT | DEFAULT 0 | 评论数 |
| `status` | ENUM | NOT NULL | 状态：DRAFT/PUBLISHED |
| `created_at` | TIMESTAMP | - | 创建时间 |
| `updated_at` | TIMESTAMP | - | 更新时间 |

**索引**：
- `idx_author_id`：按作者查询
- `idx_status`：按状态过滤
- `idx_created_at DESC`：按时间降序排列
- `FULLTEXT idx_fulltext (title, body)`：全文搜索

### 媒体类型枚举（MediaType）

| 值 | 说明 |
|----|------|
| `TEXT` | 纯文字内容 |
| `IMAGE` | 包含图片 |
| `VIDEO` | 包含视频 |

### 内容状态枚举（ContentStatus）

| 值 | 说明 |
|----|------|
| `PUBLISHED` | 已发布，公开可见 |
| `DRAFT` | 草稿，仅作者可见 |

### ContentTag 实体

**表名**：`content_tags`（关联表）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT | 主键 |
| `content_id` | BIGINT | 内容 ID |
| `tag` | VARCHAR(50) | 标签文本 |

每条内容可关联多个标签，标签为独立行存储。

## 内容 CRUD 操作

### 创建内容

```
POST /api/contents
Authorization: Bearer <token>

CreateContentRequest {
  title: string (必填, max 200)
  body: string (可选)
  mediaUrl: string (可选)
  mediaType: "TEXT" | "IMAGE" | "VIDEO"
  status: "PUBLISHED" | "DRAFT"
  tags: string[]
}
```

**流程**：
1. 从 JWT 解析当前用户
2. 创建 `Content` 实体并关联作者
3. 批量创建 `ContentTag` 记录
4. 持久化并返回 `ContentDto`

### 查询内容列表

```
GET /api/contents?page=0&size=10
```

- 只返回 `PUBLISHED` 状态的内容
- 按 `created_at DESC` 排序
- 支持分页，返回 `PageResponse<ContentDto>`

### 查询内容详情

```
GET /api/contents/{id}
```

访问详情时自动增加 `view_count`（`ContentService.incrementViewCount(id)`）

### 更新内容

```
PUT /api/contents/{id}
Authorization: Bearer <token>

UpdateContentRequest {
  title: string
  body: string
  mediaUrl: string
  mediaType: string
  status: string
  tags: string[]
}
```

**权限校验**：只有内容作者才能更新，否则返回 403。

### 删除内容

```
DELETE /api/contents/{id}
Authorization: Bearer <token>
```

**权限校验**：只有内容作者才能删除，级联删除关联的评论和标签。

### 查询指定作者的内容

```
GET /api/contents/user/{authorId}?page=0&size=10
```

只返回该作者已发布的内容。

### 查询我的内容

```
GET /api/contents/my?status=DRAFT&page=0&size=10
Authorization: Bearer <token>
```

可选过滤状态，返回当前用户所有内容（含草稿）。

## 内容状态管理

```
创建时选择状态
  ├─ PUBLISHED → 立即公开，出现在首页 Feed
  └─ DRAFT → 仅在「我的内容」中可见

通过编辑可以切换状态：
  DRAFT → PUBLISHED（发布）
  PUBLISHED → DRAFT（撤回）
```

## 内容媒体处理

### 图片上传

```
POST /api/upload/image
Authorization: Bearer <token>
Content-Type: multipart/form-data

file: <图片文件>
```

**支持格式**：JPEG、PNG、GIF、WebP  
**存储路径**：`{upload.path}/images/{uuid}.{ext}`  
**访问 URL**：`/uploads/images/{uuid}.{ext}`

### 视频上传

```
POST /api/upload/video
Authorization: Bearer <token>
Content-Type: multipart/form-data

file: <视频文件>
```

**支持格式**：MP4、WebM、MOV  
**存储路径**：`{upload.path}/videos/{uuid}.{ext}`  
**访问 URL**：`/uploads/videos/{uuid}.{ext}`

**文件大小限制**：单文件最大 100MB

### FileUploadService 核心逻辑

1. 校验文件 MIME 类型
2. 生成 UUID 文件名（防止重名）
3. 创建目录（如不存在）
4. 写入文件到本地磁盘
5. 返回可访问的 URL 路径

## 内容标签系统

### 创建内容时添加标签

在 `CreateContentRequest.tags` 中传入标签数组，Service 层自动创建对应 `ContentTag` 记录。

### 标签搜索

通过 `GET /api/search/contents?tag=<标签名>` 搜索包含该标签的内容。

### 热门标签

`GET /api/search/tags/popular?limit=20` 返回出现频次最高的标签列表。

### 标签建议

`GET /api/search/tags/suggest?q=<前缀>` 返回以指定前缀开头的标签建议（自动补全）。

## 前端内容管理

### ContentFeed 组件

首页内容流，支持两种模式切换：
- **全部**：显示所有已发布内容
- **订阅**：显示已关注作者的内容（需登录）

### ContentCard 组件

内容卡片，展示：标题、正文摘要、媒体预览（图片/视频缩略图）、标签、作者、时间、浏览量、评论数。

### MediaUploader 组件

支持拖拽或点击上传图片/视频，上传成功后返回 URL 并预览。

### content store（Pinia）

管理当前内容详情、加载状态，提供 `fetchContent`、`createContent`、`updateContent`、`deleteContent` 等 Action。
