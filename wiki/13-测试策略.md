# 测试策略

## 测试分层

```
┌─────────────────────────────────────────┐
│           E2E 端到端测试                  │  ← Playwright
│    模拟真实用户操作，验证完整业务流程        │
├─────────────────────────────────────────┤
│          前端组件测试                     │  ← Vitest + Vue Test Utils
│    测试组件渲染、交互、状态管理             │
├─────────────────────────────────────────┤
│          后端单元测试                     │  ← JUnit 5 + Mockito
│    测试 Service 层业务逻辑、Repository 层  │
└─────────────────────────────────────────┘
```

## 前端组件测试

### 测试框架

- **Vitest**：测试执行框架（Vite 原生支持）
- **Vue Test Utils**：Vue 组件测试工具
- **@vue/test-utils**：组件挂载与交互模拟

### 测试文件位置

```
communication-frontend/src/stores/__tests__/
└── auth.test.ts
```

### 运行测试

```bash
cd communication-frontend

# 安装依赖
pnpm install

# 运行单元测试
pnpm test:unit

# 运行测试并生成覆盖率报告
pnpm test:unit --coverage

# 监视模式
pnpm test:unit --watch
```

### 测试示例（auth store）

```typescript
// stores/__tests__/auth.test.ts
import { describe, it, expect, beforeEach } from 'vitest'
import { setActivePinia, createPinia } from 'pinia'
import { useAuthStore } from '../auth'

describe('Auth Store', () => {
  beforeEach(() => {
    setActivePinia(createPinia())
  })

  it('should initialize with unauthenticated state', () => {
    const store = useAuthStore()
    expect(store.isAuthenticated).toBe(false)
    expect(store.user).toBeNull()
  })

  it('should update auth state on login', () => {
    const store = useAuthStore()
    store.token = 'test-token'
    store.user = { id: 1, username: 'test' }
    expect(store.isAuthenticated).toBe(true)
  })
})
```

## 端到端测试

### 测试框架

- **Playwright**：跨浏览器 E2E 测试
- **@playwright/test**：测试框架

### 测试配置

`playwright.config.ts`：

```typescript
import { defineConfig } from '@playwright/test'

export default defineConfig({
  testDir: './e2e',
  use: {
    baseURL: 'http://localhost:5173',
    trace: 'on-first-retry',
  },
  projects: [
    { name: 'chromium', use: { browserName: 'chromium' } },
    { name: 'firefox', use: { browserName: 'firefox' } },
  ],
})
```

### 运行测试

```bash
cd communication-frontend

# 安装 Playwright 浏览器
pnpm exec playwright install

# 运行 E2E 测试
pnpm test:e2e

# 运行特定浏览器测试
pnpm test:e2e --project=chromium

# 以 UI 模式运行（调试）
pnpm test:e2e --ui
```

### 核心 E2E 测试用例

| 测试场景 | 步骤 |
|---------|------|
| 用户注册登录 | 访问注册页 → 填写信息 → 提交 → 验证登录状态 |
| 内容发布 | 登录 → 点击创建 → 填写内容 → 提交 → 验证列表显示 |
| 评论互动 | 登录 → 访问内容详情 → 发表评论 → 验证评论显示 |
| 订阅功能 | 登录 → 访问用户主页 → 点击关注 → 验证订阅状态 |
| 搜索功能 | 输入关键词 → 提交搜索 → 验证结果列表 |

```yaml

  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

          cd communication-backend
          ./mvnw test
      
      - name: Frontend Tests
        run: |
          cd communication-frontend
          pnpm install
          pnpm test:unit
          pnpm test:e2e
```

## 测试最佳实践

1. **测试命名清晰**：`should_do_something_when_condition`
2. **独立测试**：每个测试不依赖其他测试的执行顺序
3. **Arrange-Act-Assert**：准备数据 → 执行操作 → 验证结果
4. **模拟外部依赖**：使用 Mockito/Vitest mock 隔离被测单元
5. **边界条件**：测试空值、最大值、异常输入等边界情况
