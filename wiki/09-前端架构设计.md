## Vue 3 组件系统

### Composition API

全项目采用 `<script setup>` 语法糖，统一使用 Composition API：

```vue
<script setup lang="ts">
import { ref, computed, onMounted } from 'vue'
import { useAuthStore } from '@/stores/auth'

const authStore = useAuthStore()
const loading = ref(false)
</script>
```

### 组件设计原则

- **原子化**：`CommentItem`、`ContentCard` 等小组件专注单一职责
- **容器/展示分离**：`ContentFeed`（容器，处理数据获取）+ `ContentCard`（展示）
- **路由懒加载**：所有页面视图通过 `() => import(...)` 动态导入

## Pinia 状态管理

### auth store

管理用户认证状态，Token 与用户信息持久化到 `localStorage`：

| 状态 | 类型 | 说明 |
|------|------|------|
| `token` | `string \| null` | JWT Token |
| `user` | `User \| null` | 当前用户信息 |
| `loading` | `boolean` | 请求加载中 |
| `isAuthenticated` | `computed boolean` | 是否已认证 |

**Actions**：`register`、`login`、`logout`、`fetchCurrentUser`、`updateUser`

### content store

管理当前内容详情的缓存状态，避免重复请求。

## Vue Router 路由配置

### 路由表

| 路径 | 组件 | 认证要求 |
|------|------|---------|
| `/` | `HomeView` | 无 |
| `/login` | `LoginView` | 仅游客（`guest: true`） |
| `/register` | `RegisterView` | 仅游客（`guest: true`） |
| `/profile/:username` | `ProfileView` | 无 |
| `/dashboard` | `DashboardView` | 需认证 |
| `/subscriptions` | `SubscriptionsView` | 需认证 |
| `/create` | `CreateContentView` | 需认证 |
| `/content/:id` | `ContentDetailView` | 无 |
| `/edit/:id` | `EditContentView` | 需认证 |
| `/search` | `SearchView` | 无 |
| `/:pathMatch(.*)` | `NotFoundView` | 无 |

### 路由守卫

```typescript
router.beforeEach((to, _from, next) => {
  const authStore = useAuthStore()

  // 已认证用户重定向离开 guest-only 页面
  if (to.meta.guest && authStore.isAuthenticated) {
    next({ name: 'home' })
    return
  }

  // 未认证用户访问受保护路由时重定向到登录页
  if (to.meta.requiresAuth && !authStore.isAuthenticated) {
    next({ name: 'login', query: { redirect: to.fullPath } })
    return
  }

  next()
})
```

## Vite 构建工具配置

### 路径别名

`@` 指向 `src/` 目录，所有内部引用使用 `@/` 前缀：

```typescript
// 代替
import ContentCard from '../../../components/content/ContentCard.vue'
// 使用
import ContentCard from '@/components/content/ContentCard.vue'
```

### 自动导入

使用 `unplugin-auto-import` 和 `unplugin-vue-components` 实现：
- Vue 3 组合式 API（`ref`、`reactive`、`computed` 等）自动导入
- Element Plus 组件按需自动导入

## HTTP 请求封装（api/http.ts）

### Axios 实例配置

```typescript
const http = axios.create({
  baseURL: '/api',
  timeout: 15000
})
```

### 请求拦截器

自动为每个请求添加 JWT Token：

```typescript
http.interceptors.request.use(config => {
  const token = localStorage.getItem('token')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})
```

### 响应拦截器

统一处理 401 未认证（自动登出）和错误提示。

## UI 设计规范

### 主题色变量（variables.css）

```css
:root {
  --color-primary: #FF6B35;     /* 主色：橙红 */
  --color-secondary: #F7C59F;   /* 辅色：杏色 */
  --color-background: #FFF8F0;  /* 背景：米白 */
  --color-text: #2C3E50;        /* 正文：深色 */
}
```

### Element Plus 主题定制

通过 CSS 变量覆盖 Element Plus 的 `--el-color-primary` 实现主色调适配。

### 响应式设计

- 使用 CSS Grid 和 Flexbox 布局
- 媒体查询适配移动端（<768px）与桌面端
- 内容卡片在移动端单列、桌面端双/三列布局

### 交互细节

- 卡片悬停时阴影加深动画（`transition: box-shadow 0.2s`）
- 按钮点击反馈（按压效果）
- 图片使用 `LazyImage` 懒加载，减少首屏流量
- 加载中使用骨架屏（`el-skeleton`）
- 空状态使用统一 `EmptyState` 组件
